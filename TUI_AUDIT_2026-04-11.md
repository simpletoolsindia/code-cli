## 1. Code-Edit Flow — AUDIT FINDINGS

**Current behavior**:
- File write/edit goes through `file_write` native tool (no diff, no approval)
- Agent sends `file_write` with path + content, tool executes synchronously
- No diff shown before write — file is overwritten silently
- `editInExternalEditor()` in `src/editor/index.ts` exists but **NOT integrated** into the agent loop

**Verdict**: `P0 — Critical gap`

**Evidence**:
```
src/native-tools/files.ts → fileWrite() → fs.writeFileSync() → no preview
src/editor/index.ts → reviewPatch() → NOT called anywhere
src/diff/index.ts → generateDiff() → EXISTS but NOT called in file write flow
```

**What exists but isn't used**:
- `generateDiff(oldContent, newContent, path)` — full Myers diff
- `applyDiff()` — patch application
- `reviewPatch()` — open diff in `$EDITOR`
- `PermissionServiceImpl` — approval system

## 2. Tool-Calling Visibility — AUDIT FINDINGS

**Current behavior**:
- REPL prints tool name + args inline on one line: `🔧 tool_name {"arg": "val"}`
- Then a spinner shows "Running" with elapsed time
- Then result is formatted via `renderToolResult()` and printed
- Tool results truncated to 2 lines max
- No streaming, no incremental output, no tool-level timing beyond elapsed ms

**Good**:
- Tool name + args visible before execution
- Spinner with elapsed time
- Color-coded success/failure

**Missing**:
- No expandable result (truncated at 2 lines, user can't see full output)
- No sequential tool queue — all run one after another with no clear separation
- No per-tool status beyond spinner (queued → running → done → error)
- No tool call grouping (if 3 file reads happen in one turn, they blur together)

**Verdict**: `P1 — Needs improvement`

## 3. Diff / Changeset Display — AUDIT FINDINGS

**Current behavior**:
- No diff shown in REPL at all
- `generateDiff()` in `src/diff/index.ts` is defined but never called
- Tool results show "written to: path" as plain text

**What exists but isn't used**:
- Myers diff algorithm with context lines
- Hunk-based diff output (`@@ -N,M +N,M @@`)
- Addition/removal counts
- `formatDiffStats()` for `+N -M` stats

**Verdict**: `P0 — Critical gap`

---

## 4. Command / Log Display — AUDIT FINDINGS

**Current behavior**:
- `run_command` tool output streamed directly to stdout
- stderr goes to `process.stderr` (visible but unstructured)
- No interleaving logic — all stdout
- Timing info only in spinner, not in log

**Verdict**: `P2 — Low priority`

---

## 5. Approval Flow — AUDIT FINDINGS

**Current behavior**:
- `PermissionServiceImpl` is defined in `src/permission/index.ts` with full model
- Banned commands list, safe read-only list, rule-based auto-approval
- **BUT**: The permission service is NEVER instantiated or called in the agent loop
- No `onPermissionRequest` callback wired to REPL
- `notifications.ts` has `onPermissionRequest` hook but it's not connected to file writes

**What exists but isn't used**:
- `PermissionServiceImpl` with `request()`, `isAllowed()`, `recordDecision()`
- `SAFE_READONLY_COMMANDS` allowlist
- `BANNED_COMMANDS` blocklist
- `createPermissionRequest()` factory
- 4 decision types: `allow`, `deny`, `allow_session`, `ask`

**Verdict**: `P0 — Critical gap` (full permission system defined but offline)

---

## 6. Browser / Preview Experience — AUDIT FINDINGS

**Current behavior**:
- `open_in_browser` tool exists (executes `open`/`start`/`xdg-open`)
- No in-terminal preview of rendered HTML/Markdown/JSON
- No embedded browser (Browsh-like or WebView)

**Verdict**: `P1 — Planned for Phase 7`

## Architecture Summary

```
REPL Mode (src/index.ts)
├── Header (renderHeader)
├── REPL Loop (readline)
│   ├── @agent detection
│   ├── Memory injection
│   ├── Agent Loop (createProvider)
│   │   ├── Tool Calls (executeTool)
│   │   │   ├── tool_name + args printed inline
│   │   │   ├── spinner (FunSpinner)
│   │   │   └── renderToolResult() output
│   │   └── Response (streamText → panel)
│   └── /commands (/switch, /tools, /agents, etc.)

├── NOT WIRED: Diff generation before file_write
└── NOT WIRED: editor/reviewPatch()

Ink TUI (src/ui/ink/)
├── StatusBar, Header, Body, Input, ToolPanel, Spinner, Tips
└── Uses useAgentLoop hook (exists but limited)

├── Basic terminal-kit layout
└── Falls back to REPL on errors
```

## What's Actually Good

1. **Fallback chain** — YouTube transcript has 5-layer fallback (Python API > RapidAPI > Invidious > Transcription.com > yt-dlp)
2. **Spinner system** — FunSpinner with phase states, elapsed time, progress bar
3. **Tool routing** — `renderToolResult()` routes to specific formatters per tool type
4. **Color system** — Full theme with foreground, accent, muted, semantic colors
5. **Agent system** — Custom agents with memory, context, preferences
6. **Provider abstraction** — 45+ providers with unified API
7. **MCP integration** — TCP transport for extra_skills_mcp_tools

---

## Recommendations

1. **Wire PermissionServiceImpl into the agent loop** — especially for `file_write`
2. **Add diff display before file writes** — use existing `generateDiff()`
3. **Integrate `reviewPatch()` as approval flow** — use `$EDITOR` for review
4. **Expand tool results** — allow expanding truncated tool output
5. **Add sequential tool visibility** — show tool queue/progress state
6. **Add streaming token display** — show tokens as they arrive
